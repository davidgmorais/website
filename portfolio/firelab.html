FIRELab

Wildfire Imagery Analysis & Processing

_Year = 2021

_Context = Informatics Degree Final Project

_Project-type = Group

From vegetation characterization to fire segmentation and progression, and much more, FIRELab helps environmental researchers, data scientists and engineers streamline their work.

All this in a responsive, easy-to-use, all-in-one platform.

The goal of the project was to develop a set of tools as well as implement and adapt already developed ones and aggregate them in a ready-to-use web application of processing, support, analysis and visualisation of data such as forest fire propagation and gas emission to the atmosphere.

The fixed problem

Great part of the work in terms of visualisation and characterization of fire areas were being done manually by the engineers, resulting in a slow, repetitive and fastidious process which could be prone to error.

Additionally, segmenting the burn area from a video or images was a challenging task due to the erratic behaviour of flames and heavy smoke obstruction of crucial parts of the frame.

The majority of the tools developed at the time were all CLI based, which resulted in only people with some knowledge or training in the technological area using them with ease and proficiency.

With this in mind, the goal of the work centred its attention on improving the already implemented and existing tools as well as creating new ones which consequently eased the effort made by specialists by streamlining and automating the process as a whole.


This tool can streamline the classification process, employing the automation algorithm, however it does not relinquish the possibility of manually classifying or refining the result.

Vegetation Classification

An automated classification tool was developed, which takes some user input in the form of the classification of the vegetation in one or more tiles and uses the Euclidean distance between the average colour of a classification and the average colour of a given tile to predict the values for the vegetation classification in that tile.

The web application also allows exporting the classified grid in the format of a fuel map that is compatible with third-party apps such as FARSITE.


Segmentation of areas of interest

The tool allows for extraction of frames related to areas of interest from either images or videos. The segmentation module employs a watershed algorithm which takes user clicks and drags as interactive inputs. The resulting polygon represents the contour of the segmented area and is only generated upon user confirmation.

Holes and gaps in the segmentation are ignored by applying the morphological operations of dilation and erosion, in this order.


Georeferencing & Map Visualisation

Georeferencing works by selecting points on an image and assigning them a latitude and longitude. By finding the homograph of these points, we can obtain the pixel to geographic positions relation which is then used to enable the creation of a transformation matrix that will convert any point of that image into the corresponding latitude and longitude. Applying this transformation to the polygon obtained from segmentation will make it possible to show it on a map and overlap it on an area of interest.

FIRELab's web application is capable of generating animations containing the various polygons that map the fire evolution, hence the user will be able to easily evaluate and validate the data that FIRELab's software processed from raw, unprocessed imagery captured by the drones.

This data can be exported in formats recognized by third-party applications such as DISPERFIRE, where the burning area and the time when each zone started burning is generated in the form of a matrix and can be exported automatically by the tool developed.


FIRELab is a software application that can aid in the processing of fire-related images and videos.

FIRELab was aimed to support the MoST project by enhancing existing tools as well as developing new valuable ones that enhance both outcomes and the overall user experience and performance.

Despite several obstacles along the development process, the team feels optimistic with the completed work and the expertise gained via the development of this project.

Of course the project is not without flaws and the team recognized the limitation of the work, analysing it and proposing a set of future work to be done, if the project and subsequent investigation is to be continued.

Additionally, the work was developed alongside a bigger team than any of us ever worked with and, personally, it helped me develop some soft-skills in regard to communication and problem-solving as a team.
