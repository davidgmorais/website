MAIS

Multimodal Authentication Integration System

_Year = 2022

_Context = Biometrics

_Project-type = Group

The aim of the project was to capture an item of biometric data from a person, which can be a photo of their face or a clip of their voice and then compare it to the biometric data of several other people kept in a database.

User friendly biometric system used for authentication that runs locally

Our goal was to develop a user-friendly biometric system that includes some of the most prevalent biometrics identification methods, such as face ID and voice recognition.

Two modal authentication system based on

Face authentication, which was preceded by face detection from an image, then extraction of specific features that are used by dedicated algorithms. The results or extracted features are stored in a database which is referenced at each user authentication attempt by performing the verification process, in a 1:1 relationship.


Voice-based authentication, in which a voice recording undergoes, among other things, filtering, signal alignment and removal of redundant data in order to extract features to be used by a dedicated algorithm. The results or extracted features are stored in a database which is referenced at each user authentication attempt by performing the verification process.

Data Protection

All data entered during the registration and user authentication process are encrypted and stored safely. Furthermore, implements a dashboard to display alerts/logs of how the system is used.


When an user is authenticated successfully, the model is improved by training it with the frame used in the authentication.

During the test stages of the project some limitations were recognized and discussed, such as the effect that the lighing has on the system, glasses hindering the success rate due to reflection and high quality true video being able to bypass the liveness detection.

Face ID

The Face ID module was developed using OpenCV and can detect whenever a face is present in the frame. Contains safety mechanisms to prevent multiple people being present in the image at the same time as well as a liveness detector that checks for eye blinks, to ensure that a picture of bust is not used.

The process uses the Local Binary Pattern Histogram algorithm to enroll and authenticate an user, where the former consists on training the algorithm with collected data in order to execute the latter.


Known limitations of this module include the pronunciation inherent to different accents making it harder for the software to detect the secret passphrase being said, as well as the presence of background noise and other voices interfering with the user's.

Voice Recognition

Developed using PyAudio and extracts features from audio clips where the user repeats the prompted secret words, being a text dependent authentication.

The process uses a Gaussian Mixture Model which aggregates the features into clusters and trains the model based on the number of arbitrary clips the user uses as input. The authentication is performed through the logarithmic likelihood of the features matching the stored model, where the threshold can be adjusted and has a direct impact of the False-Positive and False-Negative rates.


Authentication system based on face and voice

MAIS was developed to support a multi-modal system which implements multiple biometric authentication methods, in such a way that in order for a user to successfully authenticate itself they must complete both authentication challenges, in resemblance to a two-factor authentication system.

The main obstacles on the development of the work was the initial ideal requiring some tinkering, where a external system running on a raspberry pi as a server was altered in favor of the authentication system running locally. Furthermore, and after the project delivery, we consider that the goals were met however the optimal performance of the system is found in an isolated and controlled environment, due to lightning and noise issues.
